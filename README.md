# Availability

Share a poll link with friends so they can mark yes, no, or maybe for each event you create. Supports 1 to 30 events per poll.

**Repo:** https://github.com/jodha/availability

## Features

- Separate admin login at `/admin`
- Invite-only user signup
- Email and password login with display names
- Forgot password via email reset link
- Admin can reset any user's password
- Users see everyone's availability
- Users can update answers anytime
- Email calendar file for Yes and Maybe events
- Single timezone: configured in `.env` (default `America/Denver`)

## Quick start on Ubuntu 24.04

```bash
git clone https://github.com/jodha/availability.git
cd availability
bash scripts/ubuntu-setup.sh
nano .env
bash scripts/start-app.sh
bash scripts/start-tunnel.sh
```

After cloudflared prints your URL, set `BASE_URL` in `.env` and restart:

```bash
docker compose up -d
```

## Scripts

| Script | Purpose |
|--------|---------|
| `scripts/ubuntu-setup.sh` | Install Docker and create `.env` |
| `scripts/setup-env.sh` | Create `.env` with generated `SECRET_KEY` |
| `scripts/start-app.sh` | Build and start Docker container |
| `scripts/start-tunnel.sh` | Install cloudflared if needed and start tunnel |

## Configure `.env`

Required values:

- `SECRET_KEY` — auto-generated by `setup-env.sh`
- `ADMIN_USERNAME` and `ADMIN_PASSWORD` — your admin login
- `INVITE_CODE` — code you share with friends for signup
- `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM` — Gmail and app password
- `BASE_URL` — your public URL (trycloudflare.com or `http://UBUNTU_IP:8000`)

## Admin workflow

1. Open `BASE_URL/admin`
2. Log in with admin credentials
3. Create a poll
4. Add events (title, start time, location) — up to 30 per poll
5. Share the site link and invite code from the admin page
6. View the availability matrix
7. Reset a user's password from the Users section if needed

## Local development (macOS or Ubuntu)

```bash
cd availability
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Run tests:

```bash
pytest
```

## Ubuntu 24.04 manual setup

### 1. Install Docker

```bash
bash scripts/install-docker.sh
```

Or manually:

```bash
sudo apt update
sudo apt install -y docker.io docker-compose-v2 git
sudo systemctl enable --now docker
sudo usermod -aG docker "$USER"
newgrp docker
docker ps
```

### 2. Clone and configure

```bash
git clone https://github.com/jodha/availability.git
cd availability
bash scripts/setup-env.sh
nano .env
```

### 3. Start the app

```bash
bash scripts/start-app.sh
```

Or:

```bash
docker compose up -d --build
```

### 4. Expose with Cloudflare Tunnel

```bash
bash scripts/start-tunnel.sh
```

Copy the full `https://....trycloudflare.com` URL from the terminal output.

### 5. Same Wi-Fi testing (no tunnel)

On Ubuntu:

```bash
hostname -I
```

On another device on the same network, open `http://UBUNTU_IP:8000`.

## Data

SQLite database is stored in the Docker volume `availability_data` at `/data/availability.db`.

User passwords are stored as bcrypt hashes and cannot be looked up. Use forgot-password or admin reset instead.
