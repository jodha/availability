# Planning Rules — Availability Project

When gathering requirements or clarifying features for this project:

1. **Interview one question at a time** — ask a single focused question, wait for the answer, then ask the next.

2. **Keep the plan updated** — after each answer, update [.cursor/plans/Availability Status Plan.plan.md](.cursor/plans/Availability Status Plan.plan.md) with the new decision.

3. **Do not start coding** until you explicitly say to execute (e.g. "go ahead", "execute the plan").

4. **Summarize decisions** in the plan interview table as answers are collected.

Cursor rule file (`.cursor/rules/planning-interview.mdc`) will be added when implementation begins.
